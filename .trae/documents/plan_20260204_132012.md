# خطة استكمال نظام التذاكر (QR & Notifications)

البيانات في الصور تبدو ممتازة! ✅ لقد قمت بربط قاعدة البيانات (Postgres) والـ Redis بشكل صحيح.

## ⚠️ ملاحظة هامة بخصوص Railway
يوجد متغير واحد مهم لم يظهر في القائمة، وهو **`SERVER_URL`**.
*   **لماذا نحتاجه؟** عشان الـ API يقدر ينشئ روابط للصور والملفات بشكل صحيح.
*   **القيمة:** قم بإضافته واجعل قيمته الدومين الخاص بالتطبيق على Railway (مثلاً: `https://evolution-xxx.up.railway.app`).

## خطة التنفيذ في Next.js (مشروعنا)
بمجرد أن يعمل الـ API، سنبدأ في برمجة نظام التذاكر كالتالي:

### 1. تثبيت المكتبات
*   `qrcode`: لتوليد صور الـ QR.
*   `nodemailer`: لإرسال الإيميلات.

### 2. صفحة التحقق (Check-in Page)
*   المسار: `app/admin/verify/[bookingId]/page.tsx`
*   الوظيفة: عند مسح الـ QR، تفتح هذه الصفحة لتعرض بيانات الزائر وزر لتأكيد الحضور (Mark as Attended).

### 3. خدمات الإشعارات (Services)
*   `lib/whatsapp.ts`: للاتصال بـ Evolution API وإرسال التذكرة.
*   `lib/email.ts`: لإرسال إيميل التذكرة بتصميم HTML احترافي.
*   `lib/qrcode.ts`: لتوليد رابط التحقق المشفر.

### 4. التفعيل (Webhook)
*   تعديل `api/payment-webhook` ليقوم باستدعاء هذه الخدمات فور نجاح الدفع.

## هل نبدأ بالتنفيذ؟
سأبدأ بتثبيت المكتبات وإنشاء صفحة التحقق بينما تنتهي أنت من إعداد الـ Evolution API.
