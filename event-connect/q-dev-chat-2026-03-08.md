## event-connect - ملخص المشروع

ده app تفاعلي للحضور في الفعاليات، مبني بـ React + Vite + Express + SQLite.

### Stack
- Frontend: React 19, React Router, Tailwind CSS, Framer Motion
- Backend: Express.js + Socket.IO (real-time)
- Database: SQLite (better-sqlite3) - local file `event.db`
- AI: Gemini API (`@google/genai`)

### الـ Features
1. Login بالموبايل - لو مش موجود بيعمل register تلقائي
2. Tasks - مهام يكملها الحضور ويكسبوا points
3. Leaderboard - live بيتحدث عبر Socket.IO
4. Q&A - الحضور يبعتوا أسئلة real-time
5. Profile - صورة + contacts اللي قابلهم
6. Admin Panel - `/admin` لإدارة tasks, leaderboard, Q&A

### Structure
```
event-connect/
├── server.ts          # Express + Socket.IO + SQLite backend
├── src/
│   ├── pages/         # Login, Home, TaskDetail, Leaderboard, QnA, Profile
│   ├── pages/admin/   # AdminDashboard, AdminLeaderboard, AdminQA
│   ├── components/    # Layout, CameraCapture, NotificationProvider
│   └── context/       # UserContext (auth state)
```

### تشغيله
```bash
cd event-connect
npm install
npm run dev  # بيشغل على port 3000
```

- بدل SQLite المحلي → `user_tasks` table في Supabase
- الـ Leaderboard يبقى persistent

## Supabase Realtime - الإجابة

**نعم، real-time فعلاً** عبر WebSockets - مش محتاج Socket.IO خالص.

```
supabase
  .channel('app_settings')
  .on('postgres_changes', { event: 'UPDATE', table: 'app_settings' }, callback)
  .subscribe()
```

يعني لما الـ admin يفتح الـ tasks → كل الـ clients بيشوفوا التغيير فوراً بدون polling.

## الـ Trigger

```sql
-- لما booking يبقى confirmed → يعمل event_user تلقائي
CREATE OR REPLACE FUNCTION create_event_user_on_confirm()
RETURNS TRIGGER AS $$
BEGIN
  IF NEW.status = 'confirmed' AND OLD.status != 'confirmed' THEN
    INSERT INTO event_users (booking_id, phone, name, field, company, linkedin_url, password)
    VALUES (
      NEW.id, NEW.phone, NEW.customer_name, NEW.industry,
      NEW.company, NEW.linkedin_url,
      substring(md5(random()::text) from 1 for 6)
    )
    ON CONFLICT (booking_id) DO NOTHING;
  END IF;
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER on_booking_confirmed
  AFTER UPDATE ON bookings
  FOR EACH ROW EXECUTE FUNCTION create_event_user_on_confirm();
```
